# Hosts Consolidation: Merging hosts/ into modules/hosts/

## Problem

Host configuration is split across two directories:

- `modules/hosts/<name>.nix` — dendritic wiring (creates flake outputs, picks modules, sets specialArgs)
- `hosts/<name>/` — traditional per-host configs (system, hardware, home-manager, WM configs)

This means two places to look, import paths crossing between them, and home-manager config defined in two separate locations (the dendritic module injects HM imports, while `configuration.nix` sets up the HM user and `home.nix` adds more config).

## Design

### New structure

```
modules/hosts/jezrien/
  default.nix                    # dendritic wiring + all home-manager config inline
  _configuration.nix             # NixOS system config (boot, hardware, services, ryk.*)
  _hardware-configuration.nix    # generated by nixos-generate-config
  _niri-config.nix               # niri window manager config
  _hyprland-config.nix           # hyprland config (currently commented out)
  secrets.yaml                   # sops secrets

modules/hosts/taln/
  default.nix                    # dendritic wiring + all home-manager config inline
  _configuration.nix             # Darwin system config
  secrets.yaml                   # sops secrets
```

Files prefixed with `_` are skipped by `import-tree` (which auto-discovers all `.nix` files under `modules/`). These files are imported explicitly by `default.nix` or `_configuration.nix`.

### `default.nix` — single entry point

Each host's `default.nix` becomes the single place that defines:

1. The flake output (`flake.nixosConfigurations.jezrien` or `flake.darwinConfigurations.taln`)
2. The NixOS/Darwin module imports list (system modules, legacy modules, dendritic modules)
3. **All home-manager configuration** — base home import, host-specific HM settings, packages, module imports, stateVersion

This eliminates the current split where home-manager config lives in both `default.nix` (dendritic module imports) and `configuration.nix` → `home.nix` (user setup, packages, sops, overrides).

### `_configuration.nix` — pure system config

Home-manager setup (`home-manager.users`, `home-manager.extraSpecialArgs`, `home-manager.backupFileExtension`) moves out of `_configuration.nix` into `default.nix`. What remains is purely NixOS/Darwin system-level config.

**Jezrien `_configuration.nix`** keeps: hardware imports, nixos-hardware modules, boot, security, networking, environment, hardware, programs, services, users, powerManagement, `ryk.*` options, system.stateVersion.

The `home-manager.nixosModules.home-manager` import also moves to `default.nix`'s modules list, since that's where HM is being configured.

**Taln `_configuration.nix`** keeps: nixpkgs overlays, users, security (touchIdAuth), nix settings, system.defaults, homebrew, system.stateVersion, nixpkgs.hostPlatform, system.configurationRevision.

The `home-manager.darwinModules.home-manager` import and `home-manager.*` block move to `default.nix`.

- `sops.secrets.homelab_ssh_private_key` → inline (sopsFile path adjusts to `./secrets.yaml`)

### Sops path updates

Three files reference `hosts/${hostname}/secrets.yaml` and need path updates:

1. **`home/default.nix`** — `defaultSopsFile = ../hosts/${hostname}/secrets.yaml` becomes `../modules/hosts/${hostname}/secrets.yaml`
2. **`modules/misc/ssh.nix`** — `sopsFile = ../../hosts/${hostname}/secrets.yaml` becomes `../hosts/${hostname}/secrets.yaml` (one level up from `modules/misc/` to `modules/`, then into `hosts/${hostname}/`)
3. **Jezrien's HM block** — `sopsFile` for `homelab_ssh_private_key` becomes `./secrets.yaml` (sibling file)

The `.sops.yaml` creation rules use loose regexes (`path_regex: jezrien/secrets.yaml$`) that match any path ending in `jezrien/secrets.yaml`, so the new paths are compatible without changes.

### Import path changes in `_configuration.nix`

Jezrien's `_configuration.nix` currently imports `./hardware-configuration.nix` and `./niri-config.nix`. These become `./_hardware-configuration.nix` and `./_niri-config.nix`.

### Out of scope

The nixy host (NixOS LXC test container) was removed in a prior commit and has no files to migrate.

### Deletions

- `hosts/` directory — removed entirely

## Principles

- **One directory per host** — everything in `modules/hosts/<name>/`
- **One entry point for home-manager** — `default.nix` owns all HM config
- **Underscore = private** — `_` prefixed files are implementation details skipped by `import-tree`
- **System config stays separate** — `_configuration.nix` is purely NixOS/Darwin system config, no HM
